Complessità asintotica

La complessità asintotica di un algoritmo descrive l'andamento temporale per input di grandezza n→∞.

Sequenza di Fibonacci

Per esempio, data la sequenza di Fibonacci definita come Fn​={1Fn−1​+Fn−2​​se n=1,2se n≥3​ esistono diversi algoritmi per trovare l'n-esimo numero:

BinetIncorrettoT(n)≈1S(n)≈1
RicorsivoCorrettoT(n)≈Fn​S(n)≈n

Binet

La formula di Binet lega la sezione aurea all'n-esimo numero di Fibonacci: Fn​=5​1​(φn−φ^​n) dove φ e φ^​ sono le soluzioni a x2=x+1.

Di conseguenza, l'algoritmo risulta semplicemente essere:

Fib1(int n) -> int
	return 1/sqrt(5) * (pow(phi, n) - pow(1 - phi, n))

Complessità temporale e spaziale

In questo caso, T(n)≈S(n)≈1 dato che l'algoritmo è composto da un return e l'unico dato salvato è n.

Correttezza

Per verificane la correttezza, bisogna dimostrare che: Fn​=5​1​(φn−φ^​n)

Proseguendo per induzione, i casi base sono:

  • Per n=1, 5​1​(φ1−φ^​1)=5​1​(21+5​−1+5​​)=1=F1​
  • Per n=2, 5​1​(φ2−φ^​2)=5​1​(41+25​+5−1+25​−5​)=1=F2​

Per cui va verificato il passo induttivo: Fn−1​+Fn−2​=5​1​(φn−φ^​n) di conseguenza, dati i casi base, si suppone che la formula sia valida per Fn−1​ e Fn−2​, per cui: Fn−1​+Fn−2​​=5​1​(φn−1−φ^​n−1)+5​1​(φn−2−φ^​n−2)=5​1​(φn−1−φ^​n−1+φn−2−φ^​n−2)=5​1​((φn−1+φn−2)−(φ^​n−1+φ^​n−2))​ che va trasformata in 5​1​(φn−φ^​n), cioè: {φn=φn−1+φn−2φ^​n=φ^​n−1+φ^​n−2​⇔{φn−2φn​=φn−2φn−1​+1φ^​n−2φ^​n​=φ^​n−2φ^​n−1​+1​⇔{φ2=φ+1φ^​2=φ^​+1​ e siccome φ e φ^​ sono le uniche soluzioni del sistema, la formula è verificata.

Seppur dimostrato, Fib1 rimane incorretto perchè l'imprecisione aumenta con il crescere di n.

Ricorsivo

Nel caso ricorsivo, l'algoritmo risulta essere:

Fib2(int n) -> int
	if n <= 2
		return 1
	return Fib2(n - 1) + Fib2(n - 2)

Complessità temporale e spaziale

I passaggi che fa questo algoritmo sono: T(n)={12+T(n−1)+T(n−2)​se n=1,2se n≥3​ dove 1 è per l'if/return, mentre 2 è per l'if più l'else/return.

Per trovare il valore di T(n) si può usare un albero ricorsivo Tn​. Per n=5, l'albero T5​ sarà:

Contando i pesi sui nodi dell'albero, si può ricavare il costo: T(n)=2⋅4+1⋅5=2⋅i(Tn​)+f(Tn​) dove i(Tn​) è il numero di nodi interni e f(Tn​) è il numero di foglie.

Si può quindi dimostrare che:

  1. f(Tn​)=Fn​, per induzione:

    Tra i casi base, se:

    • n=1⇒f(T1​)=1=F1​
    • n=2⇒f(T2​)=1=F2​

    Per cui, supponendo che f(Tn−1​)=Fn−1​∧f(Tn−2​)=Fn−2​ sia vero per ipotesi induttiva, allora: f(Tn​)=f(Tn−1​)+f(Tn−2​) dato che Tn​ non possiede altri sottoalberi oltre che Tn−1​ e Tn−2​. Di conseguenza: f(Tn​)=Fn−1​+Fn−2​=Fn​

  2. i(Tn​)=f(Tn​)−1=Fn​−1

    Tra i casi base, se:

    • n=1⇒i(T1​)=0=F1​−1=f(T1​)−1
    • n=2⇒i(T2​)=0=F2​−1=f(T2​)−1

    Supponendo i(Tn−1​)=f(Tn−1​)−1∧i(Tn−2​)=f(Tn−2​)−1 vero per ipotesi induttiva, allora: i(Tn​)​=i(Tn−1​)+i(Tn−2​)+1==f(Tn−1​)−1+f(Tn−2​)−1+1==f(Tn​)−1​ visto che per la precedente dimostrazione f(Tn​)=f(Tn−1​)+f(Tn−2​).

Perciò, la complessità temporale di Fib2 sarà: T(n)=2⋅(Fn​−1)+Fn​=3Fn​−2≈Fn​ e la complessità spaziale S(n)≈n.

È anche possibile che la complessità cresce più velocemente che esponenzialmente: Fn​≥22n​, ∀n≥6 infatti per induzione, il caso base per n=6⇒F6​≥23 e il passo induttivo per n>6: Fn​=Fn−1​​≥22n−1​​+Fn−2​​≥22n−2​​⇓Fn​​≥22n−1​+22n−2​≥22n​⋅2−21​+22n​⋅2−1≥22n​(≥12​1​+21​​​)≥22n​​

Iterativo con array

Sfruttando un array, si possono salvare i valori di Fn​ senza doverli ricalcolare su Fn+1​:

Fib3(int n) -> int
	int F[n]
	F[1] = F[2] = 1
	for i = 3 to n
		F[i] = F[i-1] + F[i-2]
	return F[n]

Complessità temporale e spaziale

Il numero di istruzioni eseguite sono T(n)=3+(n−3+1)+(n−3+1+1)=2n≈n dove 3 sono per l'inizializzazione di F e il return, mentre:

  • n−3+1 sul contenuto del ciclo: eseguito per i=3 e per il resto delle n−3 volte
  • n−3+1+1 sulla condizione: per i=3, per il resto delle n−3 volte e una alla fine per i=n+1

Mentre lo spazio occupato sarà S(n)≈n, dato che viene salvato Fn​ per ogni n.

Iterativo

La precedente versione si può ottimizzare ulteriormente, salvando solamente Fn−1​ e Fn−2​:

Fib4(int n) -> int
	int f1 = 1, f2 = 1, f
	for i = 3 to n
		f = f1 + f2
		f1 = f2
		f2 = f
	return f

per cui rimane che T(n)=4n−5≈n, mentre la complessità spaziale diventa S(n)≈1.
